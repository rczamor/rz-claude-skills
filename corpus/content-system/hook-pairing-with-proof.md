# Hook Pairing with Proof

## What it is
A rule that every hook one-liner in a piece must be paired with a specific proof point — a number, a named system, a real moment. The hook earns the open; the proof earns the read. Without pairing, hooks become slogans. With pairing, they become claims you can argue about over coffee.

## Why it matters
A great hook with no proof is provocation, not thought leadership. The pairing is what makes the difference between "Your AI doesn't have a model problem. It has a context problem." (slogan) and "Your AI doesn't have a model problem. It has a context problem. At Grandstage we cut retrieved docs from 50 to 4 and accuracy went up." (claim). The first gets a like. The second gets a save and a DM asking how.

## How to apply
1. For every hook one-liner in a piece, identify the paired proof within the same paragraph or the next paragraph.
2. The proof must come from the canonical inventory: Grandstage 300% growth, $0 CAC; Helm Labs $3.25M pre-launch pipeline, 14x ACV; IBM millions across product lines; Suzy 350+ enterprise customers, six-week launch; Context Layer Engine running architecture; the 2003 Boston Beer Microsoft Access platform.
3. Use specifics: numbers, named systems, decisions, or moments. Vague is not proof.
4. If a hook has no available proof, don't use the hook. Pick a different anchor for the piece.
5. Keep the proof tight. One sentence, one number, one named system. Not a paragraph.

## Examples

**Hook + proof in a Monday Hot Take:** "Retrieval without consolidation is just efficient delivery of noise. Grandstage's first version retrieved the right 50 documents and still gave wrong answers. The second version retrieved 4 and shipped. Same data. Different system."

**Hook + proof in a Wednesday Deep Dive opening:** "Your AI doesn't have a model problem. It has a context problem. At Suzy, the Decision Engine accuracy ceiling didn't move when we changed models. It moved when we changed how context was assembled — six weeks of architecture work that 350+ enterprise customers ran on within the quarter."

**Hook + proof in a Thursday Framework close:** "Expert decision-makers use less information, not more. Your AI should too. The Decision-Readiness Test cut Suzy's context size 4x without quality loss. The model didn't change. The question we asked of every signal did."

**Hook + proof in a Friday Story:** "Models get smarter every month. Context quality stays the same. The Microsoft Access platform I built for Boston Beer in 2003 was a context system. Twenty-two years later I'm still solving the same problem with better tools."

## Anti-patterns
- Hook with no proof. "Models get smarter every month. Context quality stays the same." standing alone is a slogan. Add what changed, what shipped, what the number was.
- Proof with no hook. A pure case study without an anchor reads like a portfolio entry. The hook is what makes it a position.
- Generic proof. "We saw improvements" is not a proof point. "Accuracy went from 62% to 94%" is.
