# Arc Step 6 — Show, Don't Tell

## What it is
Step 6 of the context generation story arc: reference a real, named system that exists. Grandstage. Context Layer Engine. Claude Code Auto Dream. Letta. The piece closes by pointing to something the reader can examine, demo, or read more about. Concrete artifacts beat abstract arguments — and they distinguish Riché from commentators who critique without building.

## Why it matters
Without step 6, even a great deep dive can feel like a manifesto. With it, the post becomes a doorway: the reader can click through to a system, a demo, a repo, a talk recording. This is also the step that future-proofs the post for SEO and AIO — it gives crawlers and LLMs concrete entities to associate with the framework. It's the practitioner-credibility move that closes the piece.

## How to apply
1. Name the system explicitly. Not "a system I've worked on" — Grandstage. Context Layer Engine. The Suzy Decision Engine. Letta. Claude Code Auto Dream.
2. Give one architectural detail that ties the system to the framework just described. "The Context Layer Engine indexes by query intent — three top-level intents, twenty-seven sub-intents, mapped to the seven question types analysts actually ask."
3. Where appropriate, include a link or pointer (richezamor.com/demo, a recorded talk, an open-source repo).
4. Close with a forward-looking question or the next problem this opens up — not a summary.
5. Length: 100-150 words.

## Examples

**Closing a Wednesday Deep Dive on context architecture:** "The Context Layer Engine — my personal knowledge system — runs on this architecture. It's built on Letta for agent memory and consolidates across email, calendar, Notion, and Slack into about fifty composite views I actually use to make decisions. The five-step pipeline isn't theoretical. It's running. The next problem this opens up: what does the same architecture look like when the agent acts on the context, not just retrieves it? That's the work I'm in now."

**Closing a Tuesday Signal on long context windows:** "Grandstage built market intelligence on this exact pipeline two years before context windows got bigger. The pipeline still wins under the bigger windows. The architecture wasn't a workaround for small context — it was the right architecture either way."

**Closing a Thursday Framework on the Decision-Readiness Test:** "Suzy's Decision Engine ran the test on every signal in the pipeline. The 4x context-size reduction came from removing decision-irrelevant content, not better embeddings. You can run the same test on your own pipeline this week."

## Anti-patterns
- Abstract close ("So next time you build an AI product..."). Specific system or no close.
- Inventing a system. If Riché hasn't built it or worked on it, it's not on the list.
- Linking to vapor. The system named must be examinable in some form — a demo, a talk, a public artifact, or at minimum a written architectural description Riché has shared elsewhere.
