# Profile / Website Copy Format

## What it is
The static, slow-changing copy on Riché's professional surfaces: LinkedIn About section, richezamor.com homepage hero and bio, X bio, speaker one-sheet. Different from posts and pitches because it's read by every audience segment over a long time horizon. The copy has to position practitioner-first, signal the named systems, and hint at the live demo without becoming a marketing landing page.

## Why it matters
Profile and website copy is the second click. Someone reads a Riché LinkedIn post, finds it sharp, clicks the profile. The About section has 90 seconds to convert that interest into a follow, a connection request, or a CFP shortlist. If the profile reads like a generic PM bio ("AI Product Leader passionate about innovation"), the click bounces. If it reads like a position ("I build context architecture for AI products"), the click converts. Same logic on richezamor.com — the homepage hero is the highest-leverage real estate in the entire content system.

## How to apply

Different surfaces, different lengths, same voice.

1. **LinkedIn About (300-500 words).** Long enough to walk through the position, the named systems, and the lens. Start with the position, name the proof, name the differentiator, end with what's next.

2. **richezamor.com homepage hero (50-80 words).** Two-line position statement, one-line proof, one CTA. The Context Layer Demo is the signature asset; the hero should make clear the demo exists.

3. **richezamor.com About / Bio page (300-500 words).** Expanded version of the LinkedIn About, with the longer narrative arc — the IBM-to-Helm-Labs-to-Suzy trajectory, the through-line of context architecture, and the personal grounding (cognitive science, Boston Beer 2003).

4. **X bio (160 characters).** "Context architecture for AI products. Building the Context Layer Engine. Data is not context — here's the five-step pipeline that makes it one."

5. **Speaker one-sheet (one page, ~250 words).** Title + bio + topics + proof + demo capability + how to book. Designed to be downloaded and forwarded by event organizers.

## Examples

**LinkedIn About opening (first 200 words):**

"I build context architecture for AI products — the layer between raw data and the LLM that decides what an agent actually knows.

Data is not context. Most teams pipe data into a large language model and call it RAG. The agents that result get smarter with every model release and stay equally bad at deciding what matters. The fix is a five-step pipeline upstream of the model: curate, synthesize, consolidate, prioritize, store intelligently.

I've shipped this architecture behind 300% user growth at Grandstage with $0 customer acquisition cost, a Decision Engine launch at Suzy that moved 350+ enterprise customers in six weeks, and personalization platforms at IBM generating millions in revenue across product lines. I'm currently building the Context Layer Engine — a personal knowledge system that runs the architecture — and I demo it live on stage when I speak.

My work translates cognitive science into product architecture: how expert decision-makers use less information, not more, and what that means for AI products. I started doing this in 2003 with a Microsoft Access platform for Boston Beer Company. Twenty-two years later I'm still solving the same problem with better tools."

**Homepage hero example:**

> "Context architecture for AI products.
> Five steps from raw data to a decision your agent can make.
> See the Context Layer Demo →"

**X bio:** "Context architecture for AI products. Building the Context Layer Engine. Data is not context — here's the five-step pipeline that makes it one. Ex-Suzy, Helm Labs, IBM."

## Anti-patterns
- Profile copy as job description. "Senior Product Leader at [company]." Reads like a resume. Lead with position, not affiliation.
- Marketing landing page on richezamor.com. The site is a credibility surface, not a conversion funnel. No "Request a free consultation" CTAs.
- Voice drift. The Fatal Fifteen apply to profile copy as much as to posts. No "I'm passionate about" openings, no "Let's connect" closings.
