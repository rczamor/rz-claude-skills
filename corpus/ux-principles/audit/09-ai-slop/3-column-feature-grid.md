# The 3-Column Feature Grid

## What it is
Icon-in-colored-circle + bold title + 2-line description, repeated 3× symmetrically across a section. **THE most recognizable AI layout.** If you've seen one SaaS landing page generated by an LLM, you've seen this exact module. It's the aesthetic equivalent of the four-chord pop song: it works, but it's also why everything sounds the same.

## Why it matters
This single pattern is *the* signature of AI-generated design. It tells visitors and any half-trained designer "this site was made by a model trained on Linear / Stripe / Vercel landing pages." Three columns of feature blocks rarely match the actual feature priorities (some features matter more than others — three equal-weight tiles say they don't). It's lazy IA dressed in clean typography. The question from the audit: would a human designer at a respected studio ever ship this exact module? Almost never.

## How to apply
- Detection — search the codebase / DOM for:
  - A `<section>` containing exactly 3 `<div>` children, each with: a small SVG/icon inside a rounded background (`rounded-full` + `bg-{color}-100`), an `h3` or `h4` heading, and a 1–2 sentence paragraph.
  - Tailwind class signature: `grid grid-cols-1 md:grid-cols-3 gap-8`, where each child is a card with `<Icon class="...rounded-full bg-...">`.
  - Three repeated copy blocks of similar word-count (say, 14–18 words each) — symmetry is the giveaway.
- Replacement strategies:
  - **Asymmetric features** — pick the *one* feature that matters most, build a hero block around it (image + headline + benefit), then list the others below at smaller weight. Reflect that some features carry more weight.
  - **Bento grid** — varied tile sizes (1×2, 2×1, 2×2) communicate "these aren't equal."
  - **Story-led** — replace feature tiles with a customer story or product walkthrough that demonstrates value.
- If you genuinely need 3 features at equal weight, at least vary the visual treatment: differing card aspect ratios, icons replaced with miniature product screenshots, removing the colored circle.

## Examples (BAD vs. GOOD pairs)

BAD — the universal slop module:
```jsx
<section className="grid grid-cols-3 gap-8">
  {[
    {icon: '⚡', title: 'Lightning Fast', desc: 'Built for speed and scale.'},
    {icon: '🛡', title: 'Enterprise Security', desc: 'Bank-grade encryption.'},
    {icon: '🤝', title: 'Seamless Integration', desc: 'Works with your stack.'}
  ].map(f => (
    <div key={f.title}>
      <div className="rounded-full bg-purple-100 p-4 w-16 h-16 mb-4">{f.icon}</div>
      <h3 className="font-bold">{f.title}</h3>
      <p>{f.desc}</p>
    </div>
  ))}
</section>
```

GOOD — bento with asymmetric weight:
```jsx
<section className="grid grid-cols-12 gap-6">
  <article className="col-span-8">
    <img src="/product/reviewer.png" alt="Reviewer flow"/>
    <h3>Catch design regressions before they ship.</h3>
    <p>Our reviewer scores every PR against a 10-category audit in under 30 seconds.</p>
  </article>
  <article className="col-span-4">
    <h4>Trunk Test built in</h4>
    <p>Every page checked against Krug's 6 wayfinding questions.</p>
  </article>
  <article className="col-span-4">
    <h4>WCAG AA enforced</h4>
    <p>Contrast and focus-state checks block PRs.</p>
  </article>
</section>
```

## Anti-patterns
- Believing that swapping the icon for a custom SVG fixes the slop. The *layout* is the slop.
- Six-tile feature grids — twice the slop.
- "But this is the conversion-tested layout" — possibly true 5 years ago, now noise.
